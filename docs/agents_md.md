# AGENTS.md

For information about AGENTS.md, see [this documentation](https://developers.openai.com/codex/guides/agents-md).

## Hierarchical agents message

When the `child_agents_md` feature flag is enabled (via `[features]` in `config.toml`), Codex appends additional guidance about AGENTS.md scope and precedence to the user instructions message and emits that message even when no AGENTS.md is present.

## Startup discovery summary

When the `startup_agents_summary` feature flag is enabled, Codex builds a
gitignore-aware startup tree for the current working directory and annotates each
discovered AGENTS file with `why` / `when` routing hints generated by a batched model call.
Set `startup_agents_summary_model` in `config.toml` to override the summary model. When unset,
Codex uses `gpt-5.1-codex-mini` if the active provider is OpenAI; otherwise it falls back to the
session's primary model.
